1. Introdução¶

Neste trabalho iremos analizar dados de pacientes, para classificarmos pacientes com parkinsons.

Serão utilizadas várias técnicas de classificação, assim como técnicas de transformação dos dados que permitem a redução de dimensionalidade do dado preservando alguns aspectos dele, como sua variância ou separação entre classes.

Este trabalho foi feito de forma bem modular, a ponto de ser possível reutilizar muitos dos modelos para outros trabalhos. Além disso foram utilizadas algumas bibliotecas de manipulação de matrizes e visualização de dados.

In [ ]:
class Transformer:
    
    def __init__(self):
        self.eig_vals = None
        self.T = None
    
    def transform(self, X, min_var):
        if self.T is None:
            print('Not Fit')
            return
        cs = np.cumsum(self.eig_vals / self.eig_vals.sum())
        n_components = len([x for x in cs if x < min_var]) + 1
        subT = self.T[:, :n_components]
        return np.matmul(X, subT)

class LDA(Transformer):
    
    def fit(self, X, Y):
        
        cd = to_class_dict(X, Y)
        classes = [cd[y] for y in sorted(cd)]

        m = np.mean(np.array([x for c in classes for x in c]), axis=0)
        centroids = [np.mean(c, axis=0) for c in classes]
        covariances = [cov(c) for c in classes]

        Sw = sum([len(classes[i]) * covariances[i] for i in range(len(classes))])
        Sw_inv = np.linalg.inv(Sw)
        n_m = [(c - m).reshape(1, -1) for c in centroids]
        Sb = sum([np.dot(n_m[i].T, n_m[i]) for i in range(len(classes))])

        vals, vecs = np.linalg.eig(np.dot(Sw_inv, Sb))
        idx = vals.argsort()[::-1]
        self.eig_vals, self.T = vals[idx], np.real(vecs[:, idx])

3. Modelos de classificação¶

Todos os modelos de classificação seguem o mesmo padrão, de treinar com a matriz de dados e o vetor de classes, e retornar as classes preditas passando uma matriz onde cada linha é um vetor de atributos. Este padrão é o mesmo utilizado pelos modelos das bibliotecas do Python, permitindo a utilização delas em nossos testes.

Estes modelos se encontram no arquivo "models.py" e serão reapresentados neste documento.

3.1. KNN (K - Nearest Neighbors)¶

Para o KNN, foi utilizada uma KDTree, que é um tipo de estrutura de indexação espacial, para acelerar a busca pelos vizinhos mais próximos. O retorno da busca são tuplas representando os vizinhos, cada uma com a distância para o vizinho e a posição no vetor dele, permitindo assim saber a classe desses vizinhos.

In [ ]:
class KNN:
    
    def __init__(self, k=1):
        self.k = k
        self.X = None
        self.Y = None
        self.kdTree = None
     
    def fit(self, X_train, Y_train):
        self.X = X_train
        self.Y = Y_train
        self.kdTree = spatial.cKDTree(self.X, leafsize=10)
        
    def predict(self, X):
        if self.X is None:
            raise Exception('Not trained')
        return [self.__predict_one(x) for x in X]
     
    def __predict_one(self, x):
        query = self.kdTree.query(x, k=self.k)
        if self.k == 1:
            return self.Y[query[1]]
        classes = np.array([self.Y[i] for i in query[1]])
        return np.bincount(classes).argmax()

3.3. DMC (Distância Mínima ao Centroide)¶

In [379]:
class DMC:
    
    def __init__(self):
        self.clusters = {}
        self.centroids = None
    
    def fit(self, X_train, Y_train):
        self.clusters = utils.to_class_dict(X_train, Y_train)        
        self.centroids = {k : np.mean(self.clusters[k], axis=0) for k in self.clusters}
        
    def predict(self, X):
        if self.centroids is None:
            raise Exception('Not trained')
        return [self.__predict_one(x) for x in X]
        
    def __predict_one(self, x):
        return min(self.centroids, key=lambda c : utils.distance(self.centroids[c], x))

3.4. CQG (Classificador Quadrático Gaussiano)¶

Para este classificador, a função de densidade de probabilidade para calcular a probabilidade à posteriori é uma gaussiana, logo foi criada a classe Gaussian para facilitar o manuseio do modelo.

Como pode-se perceber, para a classificação é utilizado o logarítimo da gaussiana que facilita os cálculos, além de cancelar vários termos da desigualdade para determinar a classe do dado.

In [407]:
class Gaussian:
    def __init__(self, X, naive=False):
        self.X = X
        self.n = len(X)
        self.mean = np.mean(X, axis=0)
        c = utils.cov(X)
        if naive:
            c = np.multiply(c, np.eye(c.shape[0]))
        self.update_cov(c)
        
    def singular(self):
        return np.linalg.matrix_rank(self.cov) != self.cov.shape[0]
        
    def update_cov(self, cov):
        self.cov = cov
        self.cov_inv = np.linalg.pinv(cov)
        self.cov_det = np.linalg.det(cov)
        
    def log_prob(self, x):
        z = x - self.mean
        return - 0.5 * (np.dot(z, np.dot(self.cov_inv, z)) + np.log(self.cov_det))

Podemos também perceber que é utilizada a inversa da matriz de covariância, que caso seja uma matriz singular, não possui inversa. Logo, para tentar driblar este problema podem ser utilizadas utilizadas as técnicas Pooled e Friedman.

In [ ]:
def pooled_cov(gaussians):
    return np.average([g.cov for g in gaussians], axis=0, weights=[g.n for g in gaussians])

def __friedman_for_gaussian(g, total, pooled, alpha):
    return np.average(np.array([cov, pooled]), axis=0, weights=[(1 - alpha) * g.n, alpha * total])
        
def apply_friedman(gaussians, alpha):
    total = sum([g.n for g in gaussians])
    pooled = pooled_cov(gaussians)
    for g in gaussians:
        g.update_cov(__friedman_for_gaussian(g, total, pooled, alpha))

Com isso, vemos que Pooled é um caso particular de Friedman onde alpha = 1, logo não é necessária a distinção no código deixando-o mais limpo.

class CQG:
    
    def __init__(self, naive=False, friedman_value=0.3):
        self.gaussians = None
        self.a_priori = None
        self.naive = naive

    def a_posteriori(self, x):
        return np.array([g.log_prob(x) for g in self.gaussians])
    
    def fit(self, X_train, Y_train):
        cd = utils.to_class_dict(X_train, Y_train)
        self.gaussians = [utils.Gaussian(cd[y], self.naive) for y in sorted(cd)]
        
        if any([g.singular() for g in self.gaussians]):
            utils.apply_friedman(self.gaussians, friedman_value)
            
        self.a_priori = np.array([ np.log(g.n) for g in self.gaussians ])
        
    def predict(self, X):
        if self.gaussians is None:
            raise Exception('Not trained')
        return [self.__predict_one(x) for x in X]
        
    def __predict_one(self, x):
        probabilities = self.a_priori + self.a_posteriori(x)
        return probabilities.argmax()

4. Testes¶

Para os testes, foi construída uma classe para facilitar o teste de qualquer modelo. Esta classe permite testar até modelos das próprias bibliotecas de aprendizado do python, além de gerar suas estatísticas.

In [ ]:
class Tester:

    def __init__(self, dataset, model):
        self.model = model
        self.dataset = dataset
        self.conf_matrices = []
        
    def test(self, n_tests=100, transform=None, min_transform_var=0.999):
        for _ in range(n_tests):
            X_train, Y_train, X_test, Y_test = self.dataset.get_random_train_test()
            
            if transform:
                transform.fit(X_train, Y_train)
                X_train = transform.transform(X_train, min_transform_var)
                X_test = transform.transform(X_test, min_transform_var)
                
            model = deepcopy(self.model)
            model.fit(X_train, Y_train)
            
            cm = np.zeros((2, 2))
            
            preds = model.predict(X_test)
            
            for y, pred in zip(Y_test, preds):
                cm[y, pred] += 1
            
            self.conf_matrices.append(cm)
     
    def statistics(self):
        return Statistics(self)
    
class Statistics:
    def __init__(self, tester):
        if not tester.conf_matrices:
            print('Not tested')
        else:
            self.ms = np.array(tester.conf_matrices)
            self.m_sum = self.ms.sum(axis=0)
            self.accs = np.array([m.trace() / m.sum() for m in self.ms])
    
    def acc_mean(self):
        return np.mean(self.accs)
    
    def acc_median(self):
        return np.median(self.accs)
    
    def acc_max(self):
        return np.max(self.accs)
    
    def acc_min(self):
        return np.min(self.accs)
    
    def acc_std(self):
        return np.std(self.accs)
    
    def specificity(self):
        return self.m_sum[0, 0] / self.m_sum.sum(axis=0)[0]
    
    def sensibility(self):
        return self.m_sum[1, 1] / self.m_sum.sum(axis=0)[1]
    
    def get_values(self):
        return [self.acc_mean(), self.acc_median(), self.acc_min(), self.acc_max(), self.acc_std(), self.specificity(), self.sensibility()]

5. Conclusão¶

Podemos perceber que o NN e o MLP são os melhores classificadores para este dataset, com o NN ganhando a disputa, mas existem alguns detalhes que são importantes ressaltar. Quanto à sensibilidade, o NN é sem dúvidas o melhor, já o MLP ganha de todos na especificidade.

Em um caso de diagnóstico como este, é muito melhor termos mais falsos positivos que falsos negativos, logo para este problema, NN é sem dúvidas o melhor classificador, mas há algo muito importante que precisa ser relatado. Este dataset é construído a partir de testes em pacientes, mas ao mesmo paciente são feitos mais de 1 teste. Com isso, é muito provável que o NN esteja classificando vizinhos próximos que são da mesma pessoa, que não reflete a realidade

Aqui é apresentado um teste pegando as médias dos resultados de cada paciente, já que provavelmente isso seria mais próximo de uma classificação realística.
